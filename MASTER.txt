AGENT 6 — EXPORT ENGINE
═══════════════════════════════════════════
ROLE: Isolate engineer + bottom sheet UI. This agent owns the entire
export pipeline from filter UI to share_plus delivery.

PART A — ISOLATE FUNCTION (top-level function, NOT a class method)
  Signature:
    String buildCsvMatrix(ExportPayload payload)

  This function runs in a separate Dart isolate via compute().
  It has NO access to Flutter widgets, Hive, or BuildContext.
  It receives only ExportPayload (primitives + List/Map).

  MATRIX CONSTRUCTION ALGORITHM:
    Step 1: Extract all unique dates from payload.snapshots
            Key: snapshot["timestamp"] as DateTime, truncated to date only
            Sort ascending.
            These become X-axis columns.

    Step 2: Extract all unique rolls from all snapshotData entries.
            Sort numerically (parse String roll to int for sort, re-pad after).

    Step 3: Build a lookup: Map<String, Map<String, String>>
            Outer key: roll (String)
            Inner key: date string "yyyy-MM-dd"
            Value: "A" if isAbsent==true, "P" if isAbsent==false

    Step 4: Build most-recent name per roll:
            For each roll, find the snapshot with the latest timestamp
            and use the name from that snapshotData entry.
            This is the Y-axis label.

    Step 5: Construct CSV string using StringBuffer (not + concatenation):
            Header row: "Roll,Name," + dates.join(",") + "\n"
            For each roll:
              row = roll + "," + name + "," +
                    dates.map((d) => lookup[roll]?[d] ?? "-").join(",")
              buffer.writeln(row)

    Return buffer.toString()

  PERFORMANCE NOTE: StringBuffer is mandatory for strings >1KB.
  String + String in a loop creates O(n²) allocations.

PART B — BOTTOM SHEET WIDGET: ExportBottomSheet
  Constructor: optional String? preselectedClassId

  UI:
    Dropdown: Class selector (populated from classListProvider)
              If preselectedClassId != null, auto-select on mount.
    DateRangePicker or two date fields: From / To
              Defaults: fromDate = now.subtract(Duration(days:30))
                        toDate = DateTime.now()
    "Export CSV" button:
              Shows CircularProgressIndicator when isExporting==true
              On tap: ref.read(exportFilterNotifier.notifier).runExport()

  runExport() IMPLEMENTATION (in Agent 2 notifier, called here):
    1. Query Hive snapshotBox for all SnapshotEntry objects.
    2. Filter in-memory (no Hive query operators):
         snapshot.classId == selectedClassId (if not null)
         AND snapshot.timestamp >= fromDate
         AND snapshot.timestamp <= toDate
    3. If result is empty: set exportError = "No data for selected range"
    4. Serialize filtered snapshots to List<Map<String,dynamic>>
       (all primitives — compute() cannot receive Hive objects directly)
    5. Build ExportPayload(snapshots: serialized, classTitle: title)
    6. Call: final csv = await compute(buildCsvMatrix, payload)
    7. Write csv to a temp file:
         final dir = await getTemporaryDirectory()
         final file = File('${dir.path}/AttendX_${classTitle}_${timestamp}.csv')
         await file.writeAsString(csv)
    8. Call: await Share.shareXFiles([XFile(file.path)],
               text: 'Attendance export — $classTitle')
    9. Set isExporting = false

  ADD: path_provider to pubspec (needed for temp file path)
       path_provider: ^2.1.3

VALIDATION BEFORE HANDOFF:
  [ ] buildCsvMatrix is a top-level function (not a method — compute() requires this)
  [ ] StringBuffer used, never string + in loop
  [ ] ExportPayload contains only primitives/List/Map (no Hive objects)
  [ ] Missing date for a roll uses "-" not "" (blank cells confuse Excel)
  [ ] Temp file name includes classTitle and date for user clarity
  [ ] path_provider added to pubspec
